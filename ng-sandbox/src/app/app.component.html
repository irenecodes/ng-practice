Learning Source for Template Driven Forms

Template-driven Forms:

Pros:

  • Initial simplicity is within scope for small-medium sized forms
  • Template code is cleaner and focuses only on presentation logic

Cons:

  • Can become hard to read and maintain to keep form validation rules within template. Adding more validator tags to a field or adding cross-field validations will decrease readbility and maintainability.
  • Validation logic can't be easily unit tested

Reactive Forms:

Pros:

  • Business logic in component class - can be unit tested more easily
  • Moving form model definition to component makes it easy to define form dynamically so easier to implement more advanced use cases
  • easier to create custom validator

Key difference: Reactive forms module allows to define form modal programmatically instead of declaratively via the view.
